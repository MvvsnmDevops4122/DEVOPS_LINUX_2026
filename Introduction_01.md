# LINUX

## What is Linux?

* **Linux** is an **open-source operating system (OS)** like **Windows** or **macOS**.

* An **operating system** is system software that manages computer hardware and software resources and provides services for programs.

* Linux was created in **1991 by Linus Torvalds**.

* Unlike Windows or macOS, **Linux is open-source**, which means its **source code is freely available** and anyone can **view, modify, and distribute it**.

* Linux is widely used in **servers, cloud computing, DevOps tools, embedded systems, and mobile devices (Android)** because it is **secure, stable, and high-performance**.

## Key Features of Linux

1. **Open Source:** Linux is open-source, which means its source code is freely available and anyone can view, modify, and distribute it.

2. **Secure:** Linux has strong security features and fewer vulnerabilities compared to many other operating systems.

3. **Case Sensitive:** Linux treats uppercase and lowercase characters differently.

4. **Multitasking:** Linux can run multiple programs at the same time efficiently.

5. **Multi-User:** Multiple users can access and work on the system simultaneously without interfering with each other.

## Free vs Open Source

### Free

* **Free** means you can use the software **without cost**, but the **source code is not available**.

* You cannot **see, modify, or improve the code**.

* You cannot **add your own features** to it.

### Open Source

* **Open-source** means the **source code is freely available to everyone**.

* Anyone can **see, modify, improve, or share the code**.

* You can **add your own features to Linux** and use it in your projects.

## Basic Linux Architecture / Basic Components of Linux OS

<img width="686" height="371" alt="image" src="https://github.com/user-attachments/assets/b8cdc67c-d570-4e98-9fb7-44be6c5fa604" />

* **Hardware** – Physical parts like **CPU, RAM, disk, and input/output devices**.

* **Kernel** – The **core of the OS** that communicates with hardware like **CPU, RAM, and disks**.

* **Shell** – A program that allows the user to **run commands** and communicate with the **Linux kernel**.

* **Utilities** – **Command-line tools** to manage files, monitor the system, control users, and handle networking.

* **File System** – Organizes data into **files and directories** (everything is treated as a **file**).

* **Applications** – Software like **browsers, text editors, and tools** that run on top of Linux.

## Linux Directory Structure

<img width="824" height="512" alt="image" src="https://github.com/user-attachments/assets/eedd343d-5359-4a9e-9926-921685ffce49" />

### 1. / – Root Directory

**Top-level directory** in Linux. All other directories start from here.

### 2. /root – Root User Home

Home directory of the **root (administrator) user**.

### 3. /bin – Basic User Commands

Contains essential commands used by all users like **ls, cp, mv, cat, rm**.

### 4. /boot – Boot Files

Contains files required to **boot the Linux system**, including the **kernel**.

### 5. /etc – Configuration Files

Stores **system and application configuration files**.

### 6. /sbin – System Commands

Contains **administration commands** used by root users like **shutdown, reboot**.

### 7. /lib – Shared Libraries

Contains **libraries required** for programs in **/bin and /sbin** to run.

### 8. /opt – Optional Software

Used for installing **third-party software**.

### 9. /home – User Directories

Contains home directories of normal users (example: **/home/satya**).

### 10. /tmp – Temporary Files

Stores **temporary files** created by the system or users.
